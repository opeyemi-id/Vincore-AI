# VAI-TMP-003 Case Study Template Structure
**Version:** 1.0 | **Owner:** Opeyemi Idonuagbe | **Status:** Active
**Department:** Templates | **Code:** VAI-TMP-003

---

## Purpose

Every completed Vincore AI project must produce a case study. This template defines the structure for both the long-form case study (for the website and portfolio) and the short-form version (for LinkedIn posts and proposals).

Complete within 7 days of project close. Save in:
- Long form: `09-client-projects/[Geography]/[Client]/PRJ-001/05-Close/Marketing-Assets/`
- Portfolio copy: `10-portfolio-case-studies/`

---

## Case Study Naming Convention

```
VAI-PRT-[NUMBER]_[ClientName]CaseStudy_v1.0.md
```

Example: `VAI-PRT-004_MBHBeautyCaseStudy_v1.0.md`

## Long-Form Case Study Structure

### Header

**Title:** How [Client Name] [Achieved Specific Outcome] with [System Type]

Example: "How MBH Beauty Eliminated Manual Booking and Recovered 8 Hours Per Week"

**Category:** Revenue Systems / Operations Systems / AI Advisory
**Industry:** [Industry]
**Location:** [City, Country]
**Tools used:** [List tools]

---

### Section 1: The Client (2–3 sentences)

Brief description of the business. What they do, how long they have been operating, their scale.

Do not use the client's real name without written permission. Use "a Lagos-based lash studio" if permission is not yet confirmed.

---

### Section 2: The Challenge

**Format:** CAR — start with Challenge.

Answer these questions in prose (not bullet points for the final version):

- What was the client doing manually that was costing them time or money?
- How were they managing bookings, clients, or operations before Vincore?
- What was the specific breaking point — what made them decide to fix it?
- Were there measurable symptoms? (Hours lost per week, missed bookings, no-shows, slow response time)

**Target length:** 150–200 words.

**Key stat to include if available:**
"Before the system, [Client] was spending [X hours] per week responding to booking enquiries manually — time that could have been spent [doing actual work / taking more clients / growing the business]."

---

### Section 3: The Action

**Format:** CAR — the Action.

Describe what Vincore built. Be specific. Name the components. Explain how they connect.

**Structure:**
1. What was designed first (the blueprint)
2. What was built (component by component)
3. How it was tested
4. How it was handed over

**Tools to name:** Typeform / Website / Paystack / Airtable / Zapier / Slack / WhatsApp / Gmail / Calendly — whichever apply.

**Target length:** 200–250 words.

**Example:**
"Vincore designed a fully automated booking and client management system built on [tools]. When a client visits the website and selects a service, they are taken through a booking flow that captures their details, shows live availability, and collects a deposit via Paystack — all without any manual input from the owner. The booking is instantly logged in an Airtable CRM, a confirmation is sent to the client automatically, and a reminder fires 24 hours before the appointment."

---

### Section 4: The Result

**Format:** CAR — the Result.

This is the most important section. Lead with numbers wherever possible.

**Answer these questions:**
- What changed immediately after the system went live?
- What specific metrics improved? (Response time, no-shows, hours saved, bookings per week, revenue)
- What does the owner no longer have to do manually?
- How does the business operate differently now?
- Optional: a direct quote from the client

**Target length:** 150–200 words.

**Format for metrics:**

| Metric | Before | After |
|---|---|---|
| Booking response time | [X hours] | Under 2 minutes |
| No-shows per month | [X] | [X] |
| Hours spent on manual booking | [X hrs/week] | [X hrs/week] |
| Bookings per week | [X] | [X] |

---

### Section 5: Client Quote (if permission granted)

Format:
"[Direct quote from client about the impact of the system.]"
— [Name], [Title/Role], [Business Name]

If no quote yet, write: "Testimonial pending client approval."

---

### Section 6: What This Means for Similar Businesses (1 paragraph)

Broaden the takeaway. Who else could benefit from this?

Example:
"Any beauty studio, salon, or spa currently managing bookings through WhatsApp or Instagram DMs is running the same manual process [Client Name] was running before this system. The same solution — adapted to their specific services, pricing, and workflow — can be deployed in 3–4 weeks."

---

### Section 7: CTA

"If your business is running on manual bookings and you want to see what automation looks like for your specific situation, [book a free discovery call at vincoreai.com](#)."

## Short-Form Case Study (LinkedIn Version)

**Format:** 5–8 lines. Hook → Problem → What we built → Result → CTA.

**Template:**

```
A [industry] business in [location] was managing every booking through WhatsApp DMs.

Every enquiry needed a manual reply. No reminders. No tracking. No system.

They were losing bookings they didn't even know they had missed.

We built a fully automated booking system that:
- Responds to enquiries instantly
- Collects deposits automatically
- Sends reminders before every appointment
- Logs every client in a CRM

Result: [X hours] saved per week. No-shows down. Revenue up.

This is what done-for-you automation looks like.

Want to see what it would look like for your business? Link in bio.
```

## Agent Instructions

When generating a case study using this template:
1. Pull project details from the client's `05-Close/Knowledge-Capture/` folder
2. Pull metrics from `05-Close/Project-Closure/` and `04-Execute/Testing/`
3. Use CAR structure in that exact order
4. Never invent metrics — use "pending measurement" if data not yet available
5. Generate both long-form and LinkedIn short-form versions
6. Flag if client quote/permission is still needed
7. Add completed case study to `10-portfolio-case-studies/` and update Master Document Register
