# Delivery process

Vincore AI follows a structured 4-step process for every client engagement. This ensures consistency, reduces risk, and produces systems that last beyond the project.

---

## Step 1: Discovery and diagnosis

**Goal:** Understand the business before touching any tool.

**What happens:**
- 30–60 minute discovery call with the client
- Map current revenue flow or operations workflow
- Identify exactly where leads, time, or money are slipping through
- Assess existing tools, integrations, and data sources
- Define what "done" looks like — what does a successful outcome mean for this client?

**Output:** A written brief summarising the problem, the proposed solution, the tools involved, and the expected outcome. Client approves before we proceed.

---

## Step 2: System design

**Goal:** Build the blueprint before building the system.

**What happens:**
- Design the full automation architecture on paper/doc first
- Map every trigger, action, condition, and output
- Identify potential failure points and edge cases
- Document tool stack and integration logic
- Share the design document with the client for approval

**Output:** A system design document the client can read and approve. No surprises during build.

---

## Step 3: Build and deploy

**Goal:** Build the system exactly as designed, tested against real conditions.

**What happens:**
- Build each component of the automation system
- Test every scenario (standard flow, duplicates, errors, edge cases)
- Refine based on test results
- Deploy to the client's live tools and environment
- Final QA check before handover

**Output:** A fully working, deployed automation system in the client's own tools.

---

## Step 4: Handover and support

**Goal:** Client understands, owns, and can maintain their system.

**What happens:**
- Walkthrough call showing exactly how the system works
- Full documentation: what each step does, what to do if something breaks, how to make common updates
- Agreed number of revision rounds completed
- Post-launch support window begins (1–4 weeks depending on package)

**Output:** A confident client who understands their system and knows what to do if anything changes.

---

## Communication standards

- All project updates in writing (email or agreed channel)
- Response time: within 24 hours on business days
- Scope changes require written agreement before implementation
- No work begins without a signed scope document and deposit received
