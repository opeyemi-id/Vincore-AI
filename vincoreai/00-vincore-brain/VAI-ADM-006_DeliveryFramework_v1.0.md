# VAI-ADM-006 Vincore Delivery Framework (VDF)
**Version:** 1.0 | **Owner:** Opeyemi Idonuagbe | **Status:** Active

---

## Overview

The Vincore Delivery Framework (VDF) is the standard project delivery methodology applied to every client engagement regardless of service type or industry.

Whether building a GTM system, speed-to-lead automation, CRM, AI governance framework, executive workshop, or operations automation — the five phases never change.

---

## Phase 1: Initiate

Everything required before work officially begins.

**Subfolders:**
- Client Information — client profile, company research, industry research, decision makers, org chart
- Opportunity Documents — discovery call notes, initial meeting minutes, opportunity brief, business case
- Commercials — proposal, quotation, statement of work, contract, NDA, purchase order, initial invoice
- Kick-off — kick-off agenda, kick-off presentation, kick-off minutes, attendance register
- Governance — project charter, scope statement, stakeholder register, communication plan, project contact list

---

## Phase 2: Plan

The folder most agencies don't have. This is what makes Vincore look like a consulting firm.

**Subfolders:**
- Project Planning — project management plan, work breakdown structure (WBS), deliverables register, milestone register
- Schedule — project schedule, Gantt chart, sprint plan, weekly plan, implementation timeline, dependencies
- Resources — resource plan, team allocation, responsibility matrix (RACI)
- Risks — risk register, issue register, RAID log, assumptions, constraints
- Reporting — weekly status reports, steering committee packs, meeting minutes, action log

---

## Phase 3: Assess & Design

Merged because in consulting they are closely connected.

**Subfolders:**
- Assessment — current state assessment, AI readiness assessment, operational assessment, technology assessment, security assessment, gap analysis
- Business Analysis — requirements document, user stories, business process maps, pain point analysis, future state process
- Solution Design — solution architecture, workflow diagrams, CRM design, automation blueprint, wireframes
- Technical Design — Airtable structure, Zapier architecture, API documentation, prompt library, data dictionary

---

## Phase 4: Execute

The biggest phase.

**Subfolders:**
- Build — HTML, CSS, JavaScript, Airtable, Zapier, n8n, Claude prompts, assets, configuration files
- Testing — test plan, test cases, UAT, bug register, validation reports, client sign-off
- Deployment — deployment checklist, go-live checklist, production configuration, release notes
- Training — user manual, admin guide, SOPs, training slides, workshop materials, training recording, FAQ
- Support — support requests, enhancement requests, monthly reports, optimisation reports

---

## Phase 5: Close

What separates professionals from freelancers.

**Subfolders:**
- Project Closure — close-out report, client acceptance form, handover document, completion certificate
- Financial Closure — final invoice, payment confirmation, retainer agreement (if applicable)
- Knowledge Capture — lessons learned, internal review, project retrospective, reusable assets
- Marketing Assets — case study, before and after metrics, client testimonial, screenshots, Loom video, portfolio images, blog draft, LinkedIn post draft

---

## The Rule

Every completed project feeds the Vincore marketing engine through the Marketing Assets subfolder in Phase 5 Close.

No project is complete until the case study draft exists.
