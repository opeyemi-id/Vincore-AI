# Clients

This folder contains one subfolder per active or past client.

## Folder structure per client

```
clients/
└── client-name/
    ├── brief.md           ← Discovery notes and agreed scope
    ├── system-design.md   ← Architecture and build plan
    ├── deliverables.md    ← What was built and handed over
    ├── handover.md        ← Handover documentation for the client
    └── comms.md           ← Key communications log
```

## Naming convention

Use lowercase with hyphens: `swiftfix-hvac`, `meridian-properties`, `toronto-realty-group`

*(Client folders will be created when first paid project begins)*
